---
title: OlwibaUIContext
description: React context provider that supplies shared app state (isMobile) to the component tree.
---

<ComponentPreview name="olwiba-ui-context" />

## Usage

```tsx
import { OlwibaUIProvider, useOlwibaUI } from "@olwiba/ui";

// Wrap your app root — isMobile is auto-detected via useIsMobile() from @olwiba/cn
<OlwibaUIProvider>
  <App />
</OlwibaUIProvider>

// Override with a server-detected value to avoid hydration flicker
<OlwibaUIProvider isMobile={serverDetectedMobile}>
  <App />
</OlwibaUIProvider>

// In any child component
function Sidebar() {
  const { isMobile } = useOlwibaUI();
  return isMobile ? <MobileNav /> : <DesktopSidebar />;
}
```

## OlwibaUIProvider Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The component tree |
| `isMobile` | `boolean` | auto | Override mobile detection. When omitted, uses `useIsMobile()` from `@olwiba/cn` (768px breakpoint) |

## useOlwibaUI

```ts
const { isMobile } = useOlwibaUI();
```

| Field | Type | Description |
|-------|------|-------------|
| `isMobile` | `boolean` | `true` when the viewport is below 768px (or when overridden) |

## Notes

- When no `isMobile` prop is passed, the provider calls `useIsMobile()` from `@olwiba/cn` which reactively updates on window resize
- Pass a server-detected value (e.g. from a TanStack Start server function) to prevent hydration flicker on first render
