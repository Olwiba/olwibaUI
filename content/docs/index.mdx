## What is olwibaUI?

`@olwiba/ui` provides app-level components and hooks that extend the primitives from `@olwiba/cn` into reusable patterns for building applications.

## Package Chain

This package sits at the top of the olwiba component chain:

1. **@olwiba/cn** — Base primitives (Button, Card, Dialog, etc.) + confetti
2. **@olwiba/docs** — Documentation components (DocsSidebar, SearchDialog, etc.)
3. **@olwiba/ui** — App-level components (Spinner, PageHeader, hooks, context)

## Quick Start

```tsx
import { OlwibaUIProvider, Spinner, useMounted } from "@olwiba/ui";
import { Button } from "@olwiba/cn";

// Wrap your app root
<OlwibaUIProvider>
  <App />
</OlwibaUIProvider>

// Inside a component
function MyComponent() {
  const mounted = useMounted();
  if (!mounted) return <Spinner />;
  return <Button>Ready!</Button>;
}
```

## What's Included

### Components

- `Spinner` / `FullPageSpinner` — Loading states
- `PageHeader` — Page title with description and action slot
- `Suspensed` — React.Suspense with a default spinner fallback
- `ThemeSwitchMinimal` — Dark / light mode toggle button
- `ThemeColorUpdater` — Syncs a color theme to `data-theme` on `<html>`
- `VersionBanner` — Dismissible version announcement banner
- `RegisterHotkeys` — Declarative global keyboard shortcut registration
- `RootErrorFallback` — Full-page error boundary fallback
- `OlwibaUIProvider` / `useOlwibaUI` — Shared app state context (auto-detects isMobile)

### Hooks

- `useMounted` — Client-side hydration detection
- `useConfirm` — Headless async confirmation dialog
- `useControlledOpen` — Open/close state for modals and drawers
- `useScrolledPast` — Scroll threshold detection

### Utilities

- `fireConfetti` — Theme-aware confetti burst (re-exported from `@olwiba/cn`)
